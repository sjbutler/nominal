nominal

nominal provides a library to check Java identifier names for compliance with naming conventions. nominal provides three sets of naming conventions and a declarative language for specifying naming conventions.

Usage

The class {@code Nominal} provides four static methods that create instances using either conventions definitions packaged with the library, or a file written in nominal's language that specifies naming conventions. (Information about the language for specifying conventions is in the file docs/nominal-language in the distributed archive.) The class {@code Nominal} has a method test() used to evaluate the compliance of instances of {@code IdentifierName} with the specified set of conventions.

Instances of {@code IdentifierName} are created by the caller and require classes in idtk-j as well as {@code TokenisedName} and {@code TaggedToken}.

Requirements

nominal requires Java v8 to run.

Building

nominal can be built using the Gradle build tool. All dependencies are available from the Maven central repository or the jCenter repository. A POM is also provided so Maven should be able to build nominal.

Dependencies

Dependencies are specified in the Gradle build script and are downloaded from Maven central during the build process. If building nominal without Gradle the following dependencies are required:

  1. SLF4J - nominal uses SLF4J for logging. Incorporating SLF4J in (exisitng) code is straightforward. A copy of slf4j-api.x.x.x.jar must be on the classpath when compiling and running nominal. Another slf4j jar will also be required to support logging -- see http://slf4j.org/ for more information nominal was coded against slf4j v1.7.25, so v1.7.25 or greater should work
  2. mdsc - nominal uses mdsc v0.1.1 which is available at https://github.com/sjbutler/mdsc/
  3. idtk-j - nominal uses classes from idtk-j v0.5.0 which is available at https://github.com/sjbutler/idtk-j/
  4. ANTLR - ANTLR 4 is required to run nominal and to compile any changes to the grammar. antlr-x.x-complete.jar must be on the classpath. nominal was coded against ANTLR v4.7 See http://www.antlr.org/ (ANTLRWorks 2 is a useful tool for working with ANTLR grammars)

Caveat

nominal is research software. As such it is subject to change and no guarantees can be made about the stability of any API. There is also no guarantee that nominal will be maintained.
